Suggested repository metadata for `mail-listener`.

## Repo name

`mail-listener`

## Tagline

Turn mailbox activity into durable events for agents, workflows, and automation.

## Short description

A provider-extensible mail listener foundation that turns mailbox activity into durable events. Ships with a validated Gmail IMAP IDLE adapter and is designed for OpenClaw-style agent workflows.

## GitHub About text

`mail-listener` is a provider-neutral mail-ingress foundation for automation and agent systems. It helps teams keep mailbox listeners running reliably, detect new mail with low latency, and emit stable events that can be consumed by OpenClaw agents, skills, and downstream workflows. The current implementation ships with a validated Gmail IMAP IDLE adapter.

## Suggested homepage/positioning notes

Use this project when you need:
- a durable mailbox listener
- a clean event boundary between email and automation
- an OpenClaw-friendly mail ingress layer

Do not position it as:
- a full email workflow platform
- a complete multi-provider product today
- a full mail parsing/attachment platform yet
